# DuckDB SQL Editor - 機能一覧

## DB管理

| 機能 | 説明 |
|------|------|
| インメモリDB | 起動時のデフォルト。ファイルを保存しない一時的な作業領域 |
| DBファイル作成 | ヘッダーの「新規作成」ボタンで `.duckdb` ファイルを `db-files/` フォルダに作成 |
| DB切り替え | ヘッダーのドロップダウンで切り替え。テーブル一覧・保存済みクエリが自動更新 |
| DB一覧更新 | 「↺」ボタンで `db-files/` フォルダを再スキャン |

## ファイル読み込み

| 機能 | 説明 |
|------|------|
| ドラッグ&ドロップ | Parquet / CSV / JSON ファイルを左サイドバーのTABLESエリアにドロップ |
| 複数ファイル対応 | 複数ファイルを同時にドロップ可能 |
| VIEWとして登録 | ファイルはコピーされず、元ファイルへの参照としてVIEWを作成 |
| 対応形式 | `.parquet` / `.csv` / `.json` |

## テーブル一覧（左サイドバー）

| 機能 | 説明 |
|------|------|
| テーブル/VIEW一覧 | 現在のDBに存在するテーブルとVIEWを表示 |
| ダブルクリック | `SELECT * FROM テーブル名` を自動入力して即実行 |
| ホバーツールチップ | VIEWはSQL定義、テーブルはカラム一覧を表示 |
| テーブル名ドラッグ | SQLエディタにドラッグ&ドロップでテーブル名を挿入 |
| ✕ボタン | テーブル/VIEWを削除 |

## SQLエディタ

| 機能 | 説明 |
|------|------|
| SQL入力 | 複数行のSQL入力に対応 |
| クエリ実行 | 「▶ 実行」ボタンまたは `Ctrl+Enter` |
| クエリ保存 | 「保存」ボタンで現在のSQLを保存済みクエリに追加 |
| クリア | 「クリア」ボタンでエディタを空にする |

## 結果テーブル

| 機能 | 説明 |
|------|------|
| 表形式表示 | クエリ結果をヘッダー付きテーブルで表示 |
| ページネーション | 1ページ100件。上下のナビゲーションバーで移動 |
| 件数表示 | `現在件数 / 総件数 (ページ数)` 形式で表示 |
| 横スクロール | カラム数が多い場合は横スクロール対応 |

## エクスポート

| 機能 | 説明 |
|------|------|
| CSV | UTF-8 BOM付きCSVで保存 |
| JSON | JSON配列形式で保存 |
| Parquet | Parquet形式で保存 |
| 初期ファイル名 | `YYYYMMDD_HHMMSS` 形式の日時が自動セット |

## 保存済みクエリ（右サイドバー）

| 機能 | 説明 |
|------|------|
| クエリ保存 | SQLをそのまま配列として保存。名前入力不要 |
| クリックで読み込み | クリックでエディタにSQLを展開 |
| ホバーツールチップ | SQL全文をツールチップで表示 |
| ドラッグで並び替え | ドラッグ&ドロップで順序を変更 |
| ✕ボタン | クエリを削除 |
| DBごとに管理 | 保存済みクエリはDBファイルごとに独立して保存 |
| 保存場所 | `db-files/<DB名>.queries.json` / インメモリは `_inmemory.queries.json` |
