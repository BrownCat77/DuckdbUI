# DuckDB SQL Editor - 使い方

## 起動方法

```bash
uv run duckdbui
```

## 基本的な使い方

### 1. ファイルを読み込む

左サイドバーの **TABLES** エリアに Parquet / CSV / JSON ファイルをドラッグ&ドロップします。

- 複数ファイルを同時にドロップできます
- ファイルはコピーされず、元ファイルへの参照（VIEW）として登録されます
- Windowsエクスプローラーからのドロップに対応しています

### 2. SQLを実行する

エディタにSQLを入力して実行します。

```sql
SELECT * FROM your_file
```

実行方法：
- `Ctrl+Enter` キー
- 「▶ 実行」ボタン

テーブル名をダブルクリックすると `SELECT * FROM テーブル名` が自動入力されて即実行されます。

### 3. テーブル名をエディタに挿入する

テーブル一覧のテーブル名をSQLエディタにドラッグ&ドロップすると、カーソル位置にテーブル名が挿入されます。

### 4. 結果を確認する

クエリ結果は下部のテーブルに表示されます。

- 1ページ100件表示
- `◀` / `▶` ボタンでページ移動
- `1-100 / 914件 (1/10ページ)` のように件数を確認できます

### 5. 結果をエクスポートする

クエリ実行後、ボタンバー右側のエクスポートボタンが有効になります。

| ボタン | 形式 |
|--------|------|
| CSV | UTF-8 BOM付きCSV |
| JSON | JSON配列 |
| Parquet | Parquet形式 |

保存ダイアログの初期ファイル名は `20260328_161800.csv` のような日時形式です。

## DBファイルの管理

### 新しいDBを作成する

1. ヘッダーの「新規作成」ボタンをクリック
2. DBファイル名を入力（`.duckdb` は自動付与）
3. `db-files/` フォルダにファイルが作成され、自動的に切り替わります

### DBを切り替える

ヘッダーのドロップダウンからDBを選択します。切り替えると：
- テーブル一覧がそのDBの内容に更新されます
- 保存済みクエリもそのDBに紐づいたものに切り替わります

### インメモリモード

デフォルトの `(インメモリ)` はアプリを閉じるとデータが消えます。一時的な分析作業に使用してください。

## クエリの保存と管理

### クエリを保存する

1. エディタにSQLを入力
2. 「保存」ボタンをクリック
3. 右サイドバーの **SAVED QUERIES** に追加されます

### 保存済みクエリを使う

- **クリック**: エディタにSQLを展開
- **ホバー**: SQL全文をツールチップで確認
- **ドラッグ**: 上下にドラッグして順序を並び替え
- **✕**: クエリを削除

### 保存場所

保存済みクエリはDBごとに `db-files/` フォルダに保存されます。

```
db-files/
├── mydata.duckdb              # DBファイル
├── mydata.duckdb.queries.json # mydata用の保存クエリ
└── _inmemory.queries.json     # インメモリ用の保存クエリ
```

## テーブル情報を確認する

テーブル一覧のテーブル名にマウスをホバーするとツールチップが表示されます。

- **VIEW**: SQL定義（`CREATE VIEW ...`）
- **テーブル**: カラム名と型の一覧
